# Alpha 前端基线

本文档是 issue #156 的 alpha frontend baseline 交接口径。它冻结的是当前
UI/交互的核心流程，不冻结视觉 polish、响应式细节、文案微调、测试补强或小缺陷修复。
后续前端优化 issue 不应在没有单独产品/架构决策的情况下重做下列核心流程。

## 基线范围

- 首页、案例库列表、公开案例详情、点赞、筛选和搜索。
- 我的材料：全部、草稿、待审核、需修改、已通过状态视图，详情页，版本记录，审核结果显示，
  新建、修改和提交入口。
- 创建/修改：五步 wizard，Markdown 编辑与预览，AI 自查，提交确认。
- AI 自查：仅作为作者提交前建议，不阻塞正式提交；批注保持段落级、版本绑定。
- 管理员审核：审核最后提交版本，查看预览和 AI 自查意见，添加人工段落批注，
  一级审核通过或需修改。
- 公开页面：只展示公开字段和来源材料，不展示 AI 自查、人工审核批注、prompt、model 或其他
  内部审核信息。

## 冻结边界

- 前端仍以 hash-based routing 承载当前 alpha 单页应用。
- 创建/修改仍使用五步 wizard，不在普通 polish issue 中改成另一种主流程。
- AI 审核不拥有通过或退回权限，也不作为提交 gating。
- AI/人工批注保持段落级、绑定到版本快照。
- 管理员审核保持一级审核：通过 / 需修改。
- 公开案例只展示公开字段和来源材料，隐藏内部审核过程和模型/prompt 信息。

如需改变上述边界，另开产品或架构 issue 决策；不要把流程重做混入视觉、响应式、文案、
测试或小缺陷优化。

## 与 demo 视频的关系

issue #161 负责生成和归档 E2E demo videos。demo 视频可作为交接材料，但不是关闭
issue #156 的必需条件；#156 只要求 PR 或 handoff 明确引用本 alpha 前端基线。
