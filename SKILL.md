# "强国有我"上大大思政课案例库 Skill 组合 v2.0

## 概述

这是上海大学"强国有我"大思政课案例库的**智能案例处理系统**，由一个主协调器和三个专项改写器组成。用户提供任意形式素材（文本/图片/视频/网页链接），系统自动完成分类、归主题、联网补充、模板改写，生成可直接提交的案例草稿。

**触发词**：大思政课案例、强国有我案例、思政案例入库、案例改写、案例生成

## Skill 组合架构

```
shu-dasi-skill/                 ← 主协调器（本文件）
  ├── classifier.md             ← 案例内容分类器
  ├── template-sizhengke.md     ← 思政课教学案例模板改写器（附件1）
  ├── template-kechengsizheng.md ← 课程思政案例模板改写器（附件2）
  └── template-shijian.md       ← 实践育人案例模板改写器（附件3）
```

## 全流程处理步骤

### 第一步：素材采集与理解

#### 1.1 支持的素材形式

| 素材类型 | 处理方式 | 注意事项 |
|---------|---------|---------|
| **文本** | 直接读取理解 | 新闻报道、活动总结、人物介绍、学术成果介绍等均可 |
| **网页链接** | 用 `web_fetch` 抓取全文 | 支持微信文章、新闻网页、官网页面；微信链接可能被反爬，失败时转用 `web_search` 搜索文章标题后抓取 |
| **图片文件** | 用 `read_file` 读取图片，识别内容 | 提取图中文字、人物、场景等信息 |
| **视频** | 请用户提供视频文字稿或内容摘要 | 系统暂无法直接解析视频文件 |

#### 1.2 关键信息提取清单

从素材中系统提取以下信息（缺什么联网补什么）：

- **人物**：姓名、身份（职称/职务/专业/年级）、所在单位
- **事件**：什么时间、什么地点、做了什么、结果如何
- **上大元素**：上海大学师生/校友的具体参与和贡献（**必须明确点名**）
- **政策背景**：涉及的国家战略、重要政策、领导人讲话
- **数据/成效**：具体数字（覆盖人数、服务人次、获奖情况等）
- **情感/价值**：人物的动机、选择背后的价值观、可提炼的教育意义

#### 1.3 微信公众号链接特殊处理

微信公众号文章（mp.weixin.qq.com）可能出现反爬拦截，处理流程：

1. 首次尝试 `web_fetch` 直接抓取
2. 若失败（返回空内容或错误），用 `web_search` 搜索文章标题关键词
3. 从搜索结果中找到其他平台转载版本（如上观新闻、搜狐、新浪等）
4. 交叉提取信息，以原始来源为准

---

### 第二步：联网查询补充信息

素材处理后，**主动检查并联网补充**以下常见缺失信息：

#### 2.1 必查清单

| 检查项 | 何时触发 | 搜索关键词模板 |
|-------|---------|--------------|
| 政策/战略背景 | 素材涉及国家战略、重大政策 | 政策全称 + "背景" + "意义" |
| 领导人讲话原文 | 素材提及习近平讲话但未引用原文 | "习近平" + 地点/事件 + "讲话" + 日期 |
| 人物详细信息 | 素材提及人物但背景不完整 | 人名 + "上海大学" + 职称/职务 |
| 荣誉/认定信息 | 素材提及获奖但未说明全称和背景 | 荣誉全称 + "评选" + "背景" |
| 数据核实 | 素材中有具体数字但来源不明 | 核心事实关键词 + 官方来源 |
| 学校特色项目 | 素材提及上大项目但未展开 | 项目名称 + "上海大学" |

#### 2.2 联网补充规则

- **优先级**：官方网站 > 中央媒体（人民日报/新华社/央视网） > 地方主流媒体 > 其他媒体
- **交叉验证**：关键事实至少2个来源确认
- **标注方式**：在案例正文中用括号标注补充信息来源，如"（补充信息，来源：×××）"
- **参考文献**：所有联网来源必须在案例末尾以 `[文章标题](URL)，日期` 格式列出

#### 2.3 从两个案例中总结的高效搜索策略

| 场景 | 有效搜索方式 |
|------|------------|
| 查习近平讲话原文 | "习近平" + 具体地点 + 具体日期 + "讲话全文" |
| 查高校荣誉背景 | 荣誉全称 + "评选办法" 或 "授予仪式" |
| 查学校特色项目 | 项目名 + "上海大学" + site:shu.edu.cn |
| 查人物信息 | 人名 + "上海大学" + site:shu.edu.cn 或 site:shobserver.com |
| 查政策背景 | 政策名 + site:gov.cn |

### 第三步：调用分类器，判断案例类型

读取 `classifier.md`，按照其中的分类规则和决策树判断案例类型。

| 类型代码 | 案例类型 | 适用情景 |
|---------|---------|---------|
| `TYPE_A` | 思政课教学案例 | 可用于思政理论课教学，聚焦党的创新理论实践 |
| `TYPE_B` | 课程思政共享资源案例 | 体现专业课程与思政的融合，有课程载体 |
| `TYPE_C` | 实践育人案例 | 围绕社会实践、志愿服务、援疆帮扶等实践活动 |

**核心原则**：一个素材可能同时符合多种类型，应生成多个版本。判断犹豫时，优先覆盖而非遗漏。

---

### 第四步：判断主题分类

将素材归入以下四大主题范围（可多选）：

- **强国建设**：国家重大战略、科技创新、产业发展、援疆、对口帮扶
- **上海实践**：服务上海"五个中心"建设、文化建设、社会治理
- **创新发展**：学校科技攻关、学科交叉、成果转化
- **校园文明**：文明培育、文明实践、文明创建

---

### 第五步：调用对应模板改写器

根据判断的案例类型，依次调用对应的改写器：

- `TYPE_A` → 读取 `template-sizhengke.md`
- `TYPE_B` → 读取 `template-kechengsizheng.md`
- `TYPE_C` → 读取 `template-shijian.md`

每个改写器会指导你完成信息提取、模板填写和正文改写。

---

### 第六步：生成并输出案例文件

#### 6.1 文件命名规则

```
案例_[人物/项目简称]_TYPE_[A/B/C][类型名].md
```

示例：`案例_马靖萱_TYPE_A思政课教学案例.md`

#### 6.2 多版本输出

若判定为多种类型，每个类型生成独立文件，并在所有文件完成后向用户展示汇总表：

| 版本 | 文件名 | 核心角度 | 适用场景 |
|------|--------|---------|---------|

#### 6.3 信息缺失处理规则

| 缺失信息级别 | 处理方式 |
|------------|---------|
| 素材中完全未提及 | 标注 **【待定】需补充：×××**（说明需要什么信息） |
| 素材中有线索但不完整 | 尝试联网补充，成功则填入；失败则标注 **【待定】需补充：×××（已尝试联网查询，未找到可靠来源）** |
| 联网已补充 | 正常填入，在参考文献中注明来源 |
| 需人工确认的信息（如撰稿人、审核人） | 标注 **【待定】** 并说明需人工填写 |

#### 6.4 案例信息完整度评估

每个案例文件末尾附评估报告：
- **已填字段**：列出完成的部分
- **待定字段**：列出缺失项及补充建议
- **整体完成度**：高（>80%）/ 中（50-80%）/ 低（<50%）
- **优先补充项**：对案例质量影响最大的1-2项缺失信息

---

### 第七步：质量自检

在输出最终案例前，逐项检查：

- [ ] **上大元素**：案例中是否明确提及上海大学师生/校友的姓名、单位、具体贡献？
- [ ] **政治方向**：是否紧扣习近平新时代中国特色社会主义思想？表述是否准确？
- [ ] **数据具体**：是否有可量化的数据支撑？（避免"很多""大量"等模糊表述）
- [ ] **参考文献完整**：所有联网来源是否都列出了？URL是否可点击？
- [ ] **【待定】标注清晰**：每个待定项是否说明了需要补充什么？
- [ ] **字数达标**：各部分字数是否符合模板要求？
- [ ] **语言风格**：是否平实生动、有温度？是否避免了商业广告性语言？
- [ ] **结构完整**：模板要求的每个字段是否都出现了？

## 处理规范

### 上大元素识别重点

凡素材中涉及以下内容，应**重点突出并具体点名**：
- 上海大学师生、校友的姓名、单位、贡献
- 上海大学的科研成果、教学项目、社会活动
- 与上海大学相关的地点、时间、数据
- 上大特色项目（如"种子计划""海鸥计划""尚公计划""社会脊梁"精神等）

### 政治方向把关

- 确保案例坚持正确政治方向
- 紧扣习近平新时代中国特色社会主义思想
- 突出上大师生、校友的参与和贡献
- 不使用商业广告性语言
- 领导人讲话引用务必准确（联网核实原文）

### 从实际案例中积累的经验

1. **导语要有"钩子"**：第一段就要制造悬念或画面感，让读者想继续读下去
2. **人物要有"弧光"**：展现人物的转变或成长，不能只是"某老师做了某事"
3. **数据要"贴脸"**：7分钟、2元钱——具体数字比笼统描述有力得多
4. **上大元素要"点名"**：不能只说"学校"，要明确写出"上海大学""上大文学院"等
5. **参考文献要"可点击"**：每个来源都要有标题+URL，方便用户核实
6. **思考题要"接地气"**：从学生的角度设计，避免过于抽象

## 使用示例

**用户输入**：
> https://mp.weixin.qq.com/s/xxxxx 做这个案例

**系统处理流程**：
1. `web_fetch` 抓取微信全文 → 提取人物、事件、时间、上大元素
2. 检查缺失信息 → 联网补充政策背景、领导人讲话、数据等
3. 调用 `classifier.md` 分类 → 例如 TYPE_A + TYPE_C
4. 归主题 → 例如"强国建设"
5. 调用 `template-sizhengke.md` → 生成 TYPE_A 文件
6. 调用 `template-shijian.md` → 生成 TYPE_C 文件
7. 质量自检 → 修正问题
8. 输出：2个独立 .md 文件 + 汇总表
