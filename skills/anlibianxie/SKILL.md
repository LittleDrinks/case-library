# Skill: sijiao-case-library（思政案例库通用开发）

接收任意素材，依照"立德树人·课程思政·思政课程·大思政课"政策框架，系统化完成从原始素材到标准化思政案例的全流程转化，输出可直接入库的案例文档。

---

## 核心功能

- **素材评估**：判断素材可用性、时效性、受众适配度
- **思政元素提取**：四层价值坐标系 × 六大核心主题对照 × 三层深挖法
- **案例正文生成**：依素材类型匹配模板A/B/C/D
- **标准化信息卡**：完成入库存档字段
- **场景适配**：思政课程 / 课程思政 / 大思政课三类场景差异化处理
- **质量自检**：逐项核查必要条件与质量提升指标

---

## 输入格式

用户可提供以下任意一种（或多种）输入：

| 输入方式 | 说明 |
|----------|------|
| 素材文本 | 直接粘贴新闻、故事、科学报道、政策文件等 |
| 文件路径 | 提供文档路径，读取后处理 |
| 标题/关键词 | 提供事件名称（如"天问一号"），AI补全素材背景 |
| 指定方向 | 如"请开发一个关于工匠精神的理工科案例" |

**可选附加参数**（自然语言说明即可）：
- 目标学科/课程
- 目标学段
- 目标思政场景（思政课/课程思政/大思政课）
- 侧重的思政主题

## Workflow

### 第一步：素材接收与评估

1. 读取或接收素材内容。
2. 填写素材评估维度（素材名称、类型、性质、来源、时效性、受众、学科适配性）。
3. 给出"是否适合开发"的初步判断及理由。
4. 如素材存在事实性内容，用 `web_search` 验证关键信息（时间、人物、数据）。

### 第二步：思政元素系统提取

并行运行三套工具：

**工具1：四层价值坐标系扫描**
- 逐层（个人/社会/国家/人类）扫描素材的思政相关性
- 标注各层有效性（有/无）及可提炼的具体思政点
- 输出"最强切入点"结论

**工具2：六大核心主题对照**
- 对照《高等学校课程思政建设指导纲要》（教高〔2020〕3号）六大主题
- 逐项评级（强/中/弱/无）并写出联结点
- 确定主打主题（1-2个）和辅助主题（0-1个）

**工具3：素材深挖三层法**
- 表层（What）：客观事实描述
- 中层（Why/How）：行为逻辑与背景分析
- 深层（What it means）：价值根源与思政命题共鸣

### 第三步：案例正文编写

根据素材类型选择模板：

| 素材类型 | 使用模板 |
|----------|---------|
| 人物故事、历史事件、新闻事件 | 模板A（标准叙事型） |
| 科学发现、技术成就、工程案例 | 模板B（现象-本质-价值型） |
| 需比较视角、引导中国道路认同 | 模板C（比较分析型） |
| 反面案例、事故、历史错误、道德困境 | 模板D（问题反思型） |

**模板A 必含结构**：情境还原 → 冲突与抉择 → 行动与结果 → 思政映射 → 讨论问题

**模板B 必含结构**：现象层（科学叙述）→ 本质层（专业原理）→ 价值层（思政升华）→ 讨论问题

**模板C 必含结构**：对比维度设置（表格）→ 思政落点

**模板D 必含结构**：事件客观呈现 → 多元视角还原 → 原因分层分析 → **制度回应/纠错过程**（不可省略）→ 思政价值点提炼 → 讨论问题

> ⚠️ 写作禁忌：禁止说教式表达（"这充分说明了…我们必须…"），须呈现真实困境与价值张力。

### 第四步：填写案例信息卡

必填字段（全部填写）：
- 案例编号（格式：学科代码-主题代码-序号）
- 案例标题、适用课程、适用学段
- 素材类型、素材性质
- 主打思政主题、辅助思政主题
- 适用教学场景、案例模板类型、建议教学时长
- 关键词标签（3-5个）
- 素材来源与时间

### 第五步：场景适配（可选）

如用户指定了使用场景，追加以下适配说明：

- **思政课程**：标注对应思政课具体知识点；设计引导学生运用马克思主义立场、观点、方法分析的问题
- **课程思政**：填写"专业知识锚点"和"融入时机"；专业问题与思政问题须有内在关联
- **大思政课**：说明对应的现实场景/实践基地；设计"课前-课中-课后"三段联动任务

### 第六步：质量自检

逐项核查必要条件（全达标方可输出）：
- [ ] 素材来源可追溯，无虚构成分
- [ ] 思政价值点与课程专业知识有自然联结，非强行嫁接
- [ ] 至少设计2个梯度讨论问题（含价值层问题）
- [ ] 案例信息卡所有必填字段均已填写
- [ ] 负面案例已设计"制度回应/纠错过程"出口
- [ ] 行文无说教式表达

### 第七步：输出案例文档

将完整案例保存为 Markdown 文件。

**命名规范**：`思政案例_[案例标题简称]_[YYYYMMDDHHMM].md`

**保存路径优先级**：
1. 用户指定路径
2. 原素材文档同目录
3. 用户桌面：`C:/Users/zjunc/Desktop/`
4. 工作目录

## 边界情形处理

| 情形 | 处理方式 |
|------|---------|
| 素材思政意义不明显 | 先用三层深挖法追问，再用四层坐标系扫描；几乎所有真实人类活动都含价值选择 |
| 负面/敏感素材 | 按模板D处理；必须设计"制度回应出口"；在案例中标注"讨论安全感"说明 |
| 素材信息不充分 | 用 web_search 补充事实，补充内容标注[补充说明] |
| 同一素材多角度开发 | 分别按不同模板开发，用案例编号区分版本 |
| 用户只提供标题/关键词 | 先 web_search 补全素材背景，确认事实后继续流程 |
| 用户要求特定学科适配 | 参照五场景适配指南中的"各学科适配重点"表 |
